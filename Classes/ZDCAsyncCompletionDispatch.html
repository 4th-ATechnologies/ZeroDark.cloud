ZDCAsyncCompletionDispatch

@interface ZDCAsyncCompletionDispatch : NSObject

This class helps to manage a list of tuples that need to be invoked upon the completion of some asynchronous task.

In addition to storing the array of tuples, this class provides the tools needed to atomically manage the array.

  • -pushCompletionQueue:completionBlock:forKey:

    Pushes the tuple onto the the queue for the given key, and returns the NEW count of the queue.

    Generally:

    • if the returned count is 1, you should start the async task
    • if the returned count is 2+, you can ignore the request, as you’ve already started the async task
    • if the returned count is zero, you passed a nil key or a nil completionBlock

    Declaration

    Objective-C

    - (NSUInteger)pushCompletionQueue:(nullable dispatch_queue_t)completionQueue
                  completionBlock:(nonnull id)completionBlock
                           forKey:(nonnull NSString *)key;

    Swift

    func pushCompletionQueue(_ completionQueue: DispatchQueue?, completionBlock: Any, forKey key: String) -> UInt
  • -popCompletionQueues:completionBlocks:forKey:

    Pops the entire queue for the given key. You can now enumerate the lists, and invoke the completionBlocks with the result of the async task.
